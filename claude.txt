
Write a C99 single-file compiler (`main.c`) that translates a subset of K&R C to x86-64 Linux assembly (AT&T syntax, System V ABI), emitting a `.s` file to stdout. Also write a `Makefile` and a `README.md`. All three files should be packaged into a zip called `cc.zip`.

**Build:** `cc -std=c99 -Wall -o cc main.c`
**Usage:**
```
./cc input.c > output.s
as -o output.o output.s
gcc -o output output.o
```

---

**Pipeline:** lexer → parser → symbol resolution → code generation. No IR.

**Implementation:**
- Single 32 MB arena allocator, no individual frees
- Tagged union AST (`struct Node` with `NodeKind` enum, all fields inline)
- Recursive descent parser
- Lex entire file upfront into a flat token array
- First error: `fprintf(stderr, ...) + exit(1)`
- `malloc(long)` and `free(void *)` are pre-declared builtins in an implicit outer scope

---

**Types:**
- Primitives: `int`, `char`, `short`, `long`, `float`, `double`, `unsigned`
- Pointers including multi-level (`int **`)
- `void *` — implicitly converts to/from any pointer type, no arithmetic on it
- `void` as return type only
- Flat structs and unions — members are primitives, pointers, or arrays only, no nested struct/union definitions
- Arrays with literal integer size only — `int a[10]` ok, `int a[N]`, `int a[2+3]`, `int a[]` all rejected
- No `typedef`, no `enum`

**Expressions:**
- Arithmetic: `+ - * / %`
- Bitwise: `& | ^ ~ << >>`
- Logical: `&& || !`
- Comparison: `== != < > <= >=`
- Assignment: `= += -= *= /= %= &= |= ^= <<= >>=`
- Prefix and postfix `++` `--`
- Address-of `&`, dereference `*`
- Struct member access `.` and `->`
- Array subscript `a[i]`
- Function call `f(a, b)`
- Ternary `a ? b : c`
- Cast `(type) expr`
- No `sizeof`, no constant-expressions in declarator positions

**Statements:**
- `if` / `else` (dangling-else resolved to nearest `if`)
- `while`, `for`, `do-while`
- `goto` and labels
- `break`, `continue`, `return`
- No `switch` / `case` / `default`

**Storage classes:**
- `auto` (default for locals, stack allocated)
- `extern`
- `register` (accepted, treated as `auto`)
- `static` at file scope only — no `static` inside block scope

**Functions:**
- Must be declared before use via prototype or definition
- Undeclared identifier is a hard error — no implicit extern
- Prototype parameters require explicit types (named or unnamed both ok)
- `f()` and `f(void)` both mean no parameters
- No variadic functions (`...`)

**Preprocessor:** none — any `#` in source is a hard lexer error.

---

**Code generation:**
- Target: x86-64 Linux, System V AMD64 ABI
- Integer arguments in `rdi`, `rsi`, `rdx`, `rcx`, `r8`, `r9`
- Locals on the stack, frame size computed in symbol resolution, emitted as single `sub rsp, N` prologue
- File-scope variables in `.data` (initialized) or `.bss` (uninitialized)
- String literals in `.rodata`, escaped correctly (`\n \t \r \\ " \0`)
- Emit `.section .note.GNU-stack,"",@progbits` once at the top of every output file

---

Verify the compiler builds and test it by compiling and running this program, which should print `result: 42`:

```c
void printf(char *fmt, int x);

int main() {
    int x;
    x = 40 + 2;
    printf("result: %d\n", x);
    return 0;
}
```
